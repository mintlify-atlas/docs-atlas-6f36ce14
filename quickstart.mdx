---
title: "Get started with hrns"
description: "Clone, build, and chat with an AI agent in minutes. Learn how to run hrns from the terminal and embed the loop package in your own Go program."
---

hrns ships as a single Go binary with no external runtime dependencies. The steps below take you from a fresh checkout to a running agent session, and then show you how to use the `loop` package directly to build a custom agent.

## Run the default CLI

<Steps>
  <Step title="Prerequisites">
    You need Go 1.25 or later. Verify your version:

    ```bash
    go version
    ```
  </Step>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/mishankov/hrns
    cd hrns
    ```
  </Step>
  <Step title="Build the binary">
    ```bash
    go build -o hrns .
    ```
  </Step>
  <Step title="Set environment variables">
    hrns reads three environment variables at startup. At minimum you need `HRNS_KEY` and `HRNS_BASE_URL`.

    ```bash
    export HRNS_KEY=your-api-key
    export HRNS_BASE_URL=https://api.openai.com/v1
    # Optional: skip TLS verification for local endpoints
    export HRNS_SKIP_VERIFY=false
    ```

    See [Configuration](/configuration) for a full description of each variable.
  </Step>
  <Step title="Run hrns">
    ```bash
    ./hrns
    ```

    The terminal UI starts and displays the active model name.
  </Step>
  <Step title="Type a message and press Enter">
    Type any message at the prompt and press **Enter**. The agent streams its response token-by-token and shows each tool call as it happens. When the model stops requesting tools the prompt returns for your next message.
  </Step>
</Steps>

## Build a custom agent programmatically

You can import the `loop` package directly to create agents with a custom system prompt, a different tool set, or your own UI. The example below reproduces what the default `main.go` does, stripped to its essentials:

```go
package main

import (
    "context"
    "net/http"

    "github.com/mishankov/hrns/loop"
    "github.com/mishankov/hrns/openai"
    "github.com/mishankov/hrns/tools"
    "github.com/mishankov/hrns/tui"
)

func main() {
    ctx := context.Background()

    client := openai.NewClient(
        openai.WithBaseURL("https://api.openai.com/v1"),
        openai.WithAPIKey("your-api-key"),
    )

    agnt := loop.New(
        client,
        "You are a helpful coding assistant.",
        map[string]loop.Tool{
            "read_file":   tools.ReadFileTool,
            "list_files":  tools.ListFilesTool,
            "write_file":  tools.WriteFileTool,
            "run_command": tools.CommandTool,
            "web_fetch":   tools.WebFetchTool,
        },
    )

    tuiapp := tui.New()
    tuiapp.Run(ctx, *agnt)
}
```

`loop.New` accepts any value that satisfies `ChatCompletionStreamer`, so you can swap in a different HTTP client or a mock for testing without changing the rest of your code.

<Tip>
  Once the TUI is running, type `/help` to see all available slash commands, including `/model <name>` to switch models mid-session and `/new` to start a fresh conversation without restarting the binary.
</Tip>
