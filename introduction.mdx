title: "What is hrns?"
description: "hrns is a minimal Go AI agent harness with a streaming agentic loop, built-in tools, a skills system, and a terminal UI for any OpenAI-compatible API."
---

hrns (pronounced "harness") is a deliberately small Go library and CLI for building AI agents. It wires together a streaming agentic loop, a set of built-in filesystem and network tools, a skills discovery system, and an interactive terminal UI — giving you a working agent with a single binary while staying easy to extend or embed in your own programs.

## Project structure

The codebase is organised into five focused packages.

| Package | Responsibility |
|---------|---------------|
| `loop` | Core agentic loop — streams chat completions, accumulates chunks, dispatches tool calls, and repeats until the model stops requesting tools |
| `openai` | Thin OpenAI-compatible HTTP client with streaming SSE support and functional option configuration |
| `tools` | Built-in `loop.Tool` implementations: `read_file`, `list_files`, `write_file`, `run_command`, and `web_fetch` |
| `skills` | Discovers and loads skill files (Markdown with YAML frontmatter) from `~/.agents/skills` and `./.agents/skills`, injecting them into the system prompt |
| `tui` | Minimal terminal UI that reads user input line-by-line, drives the loop, and renders streamed response and reasoning chunks |
