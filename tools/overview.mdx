The `tools` package provides five ready-to-use implementations of the `loop.Tool` interface. Each tool accepts a `map[string]any` of arguments and returns a plain string result, making them straightforward to register with `loop.New` and pass to any compatible LLM client.

  <Card title="write_file" icon="file-pen" href="/tools/write-file">

  </Card>

## Tool reference

| Tool name (key in map) | Go variable | Description |
|------------------------|-------------|-------------|
| `read_file`   | `tools.ReadFileTool`   | Reads a file from the filesystem |
| `list_files`  | `tools.ListFilesTool`  | Lists files in a directory using a glob pattern |
| `write_file`  | `tools.WriteFileTool`  | Replaces the first occurrence of a string in a file (or creates it) |
| `run_command` | `tools.CommandTool`    | Runs a shell command |
| `web_fetch`   | `tools.WebFetchTool`   | Fetches content from a URL |

## Registering all built-in tools

Pass each tool variable as a value in the map given to `loop.New`. The map key is the name the LLM uses when calling the tool.

```go
agnt := loop.New(client, systemPrompt, map[string]loop.Tool{
    "read_file":   tools.ReadFileTool,
    "list_files":  tools.ListFilesTool,
    "write_file":  tools.WriteFileTool,
    "run_command": tools.CommandTool,
    "web_fetch":   tools.WebFetchTool,
})
```

<Warning>
`run_command` executes arbitrary shell commands on the host machine. Only use it with trusted LLM providers and in environments where arbitrary code execution is acceptable.
</Warning>
